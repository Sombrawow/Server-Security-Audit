## Description
 
UFW recorded multiple waves of blocked TCP connection attempts targeting port 51820
across at least four days. All observed attempts originate from the 216.180.246.0/24
address block (IPXO LLC, US), which rotates source IPs between scanning sessions —
a technique used to evade per-IP threshold-based detection and rate-limiting controls.
 
A secondary source (Scaleway, FR) contributed 12 attempts on 2026-06-18.
 
The persistent nature of this activity across multiple days and the consistent use of
rotating IPs within the same /24 block indicates an automated, large-scale internet
scanning operation rather than a targeted attack.

## Technical Analysis
 
- **Protocol mismatch:** All attempts used TCP against port 51820. WireGuard operates
  exclusively over UDP. Successful interaction with the VPN service was not technically
  possible regardless of firewall configuration.
- **Scanner fingerprint:** All blocked packets show `WINDOW=1025`. This TCP window size
  is not generated by any standard operating system (Windows typically uses 65535 or
  64240; Linux uses 29200 or similar). A value of 1025 is a known fingerprint of
  **masscan** and similar high-speed internet scanning tools, confirming automated
  origin.
- **Evasion technique:** The use of multiple IPs from the same /24 block with identical
  per-IP event counts (16 each) indicates deliberate distribution of scan volume to
  remain below per-IP rate limits and fail2ban thresholds.
- **Firewall enforcement:** All events were blocked by UFW default-deny policy before
  reaching the application layer. No packets were forwarded to the WireGuard process.
- **Apache logs verified:** No requests from 51.159.125.200 were found in
  `/var/log/apache2/access.log`, confirming UFW enforcement was effective against
  the Scaleway IP despite its "Web App Attack" categorization in AbuseIPDB.
- **No post-compromise indicators:** No authentication attempts, exploitation activity,
  privilege escalation, or lateral movement were observed across any log source.

## Conclusion
 
This incident represents persistent, automated reconnaissance targeting the publicly
reachable WireGuard port of this host. The rotating-IP pattern within 216.180.246.0/24
across multiple days confirms an ongoing scanning operation, not an isolated probe.
 
No indicators of compromise were identified. The protocol mismatch (TCP vs UDP) meant
that no interaction with the WireGuard service was possible. UFW and the VPN-first
architecture functioned as designed.
 
The host's exposure is limited to port 51820/UDP. All other services are inaccessible
from the internet, confirmed by external port scan (nmap -sS from external host,
2026-06-18).
